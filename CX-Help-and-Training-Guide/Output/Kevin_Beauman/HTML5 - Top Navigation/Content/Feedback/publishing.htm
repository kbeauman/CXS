Publishing a Survey

Create A Project > Creating a Feedback Survey > Configure a Survey > Publish a Survey

Going live with a survey is a two-step process where you first migrate the survey to a sub-environment, such as Staging or QA. This allows you the opportunity to review and test the survey before going live. Even if you do not have such an environment, you must perform this initial step. Then publish to Production to display the Feedback Badge on your web site. You may also set up Alerts from this page, which is optional.

Before publishing, verify each web page from which the survey can be launched contains the Customer Defined code snippet.

The Publish tab offers an optional code snippet for each environment which must be embedded into each of your website templates after you publish. There is also a JavaScript Call Function in the Launch This Survey section to launch the survey from a menu, button, or other user interface element other than the Feedback Badge.

The Publish page can be accessed from the Projects page. Click the Edit icon () of the desired project and then click Publish.

When a project has revisions which have not been published, the Edit icon is replaced with an alert icon (), which functions the same way.

The Publish page is organized into two sections:

  • Publish Your Project - Contains This project is, which turns the Feedback Badge Off or On, the publishing buttons, and a link to the Alerts page. You must complete the publishing process before this setting takes effect.
  • Embed the code - This syntax must be added to every live site page where the survey is to be accessible from.

To publish a survey:

  1. Click the This project is toggle to On.
  2. Click the Publish button in Step 1. Even if there is no staging environment (server), this is a necessary first step. When the process completes, the Publish button in Step 2 is enabled.
  3. Select all syntax in the Staging Embed Snippet pane and paste the syntax into the <header> section of each web page in Staging to test the survey settings.
  4. Once testing is complete, click Publish to Production. When the process completes, a time stamp displays and Publish to Production becomes disabled.
  5. Select all syntax in the Production Embed Snippet pane and paste the syntax into the <header> section of each live web page.
  6. Go to the targeted web page(s) and verify the badge displays according to your settings. If the page was open before you published, you may need to refresh the page.

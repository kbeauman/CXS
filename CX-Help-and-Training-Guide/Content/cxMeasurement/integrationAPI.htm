What Is It?

cxIntegrate - API is a unidirectional integration at the survey respondent level. This integration product deals specifically with our API , not the specific Omniture API or Google Analytics/Universal Analytics API solution. Our Public API allows clients to programmatically access their ForeSee data via a secure, direct connection to the Data Warehouse. It provides access to both respondent and aggregate level metrics

Why Use It?

By using our API, clients are able to pull our Voice Of Customer data into their internal data bases for strategic purposes. Also, by using our API, clients get a more automated transfer of data in comparison to using a more manual SFTP.

General Information

  • Must purchase Executive Portal to gain access to the API. Exceptions require Sales Leadership/Sales Engineering approval.
  • This is a self-service solution - clients should consult with their internal technology teams familiar with API and direct them to the ForeSee Developer Portal for guidance: http://developer.foresee.com/docs-articles/public-api/

Setup Process

Step 1:

  • Determine if the client is already an Executive Portal subscriber. There is a prerequisite that the client have EP to use API.
    • If the client does not use EP, notify the Account Manager.
    • If the client is an EP user, point them to the Public API section of the ForeSee Developer Portal. This should answer the majority of questions they may have about the API.

Step 2:

  • You must provide the client with access to the API.
    • To set up authorization, you must submit a Delivery Support Ticket requesting a Portal user account be created for public API access and a key/secret for authentication to the API that you then give to the client so they can get started.

This is a self-service solution - once these two steps have been completed, it is up to the client and their technical teams familiar with API and the internal databases to program the datafeed processes.

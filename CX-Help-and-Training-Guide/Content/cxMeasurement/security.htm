Data Masking

  • Once data is masked it cannot be unmasked.
  • The following sensitive data is masked:
    • Social security number (SSN), Tax ID - 9 Digit number.
    • Credit/debit card account number - 15 or 16 digit number.
  • Open-end data masking applies to the following applications:
    • Survey (including mobile)
    • Store study surveys
    • Survey support links
    • Feedback
    • EXDL
    • IVR transcription of open ends (we get this data through FTP from our partners)
  • Masking does not apply to data collected via the following:
    • Sweepstakes through store landing pages (e.g., for drug stores )
    • Smart Thank You Pages
    • Mobile OnExit
  • Replay clients MUST fill out a Secure Entry Form (SIF) to determine which information is to be masked and which to be unmasked. By default, if a client doesn’t specify masked or unmasked for a specific information, ForeSee masks that information. Contact legal for the SIF.
  • Clients do not need to fill out a SIF for surveys to mask sensitive information in open-end responses. The system looks for patterns, such as 15 or 16 digit number strings, and automatically assume it is a credit card number and mask it, or a nine digit number string, and automatically assume it is a SS number and mask it.
    • All situations that the system believes matches the pattern are masked, whether it was sensitive information or not.
      A nine digit number string that is not actually a Social Security number is masked anyway under the assumption that it is.

Password Security

Portal Password

  • The following password criteria is applicable now for AAP, EP, and VOC.
    • Password must contain eight characters.
    • Password must contain at least one upper and one lower case alpha character.
    • Password must contain at least one number and special character.
    • Password must not contain more than two repeating characters.
      Passsword123!
    • Password must not match your username.
    • Password cannot be one of your previous six passwords.
    • Certain common phrases are not allowed.
  • Profile passwords expire every 90 days. Users receive a notification email 15 days before the password is set expire.
  • Analysts and users from FedRAMP clients must use two factor authentication (2FA), rather than relying on a password alone.
  • With regards to portal, users are provided with a single-use link as well as a single-use token for initial account setup. Upon using these one-time items, they are prompted to create a unique, private password.
    • This is applicable for both internal and external Portal users.

Account Disabling

  • User accounts are disabled for users who do not log into the portal for 90 days or more.
    • This is applicable to both internal and external portal users.
  • External users must contact ForeSee to enable their portal profile again.
    • Internal users who have their accounts disabled due to inactivity must contact DOT to re-enable their access.
  • Satisfying the 90 day requirement works by logging into either portal. So if a user logs into the EP then the 90 day count resets for AAP also, and vice versa.

Client Data Security

  • We have an Acceptable Use Policy as well as Paper and Electronic Media Policies. It prohibits users from misusing client data.
  • Employees go through annual training and must sign confidentiality agreements.
  • Printed reports containing restricted data are to be physically retained, stored, or archived only within secure Foresee office environments, and only for the minimum time deemed necessary for their use. Printers do maintain logs of who printed data.
  • We typically disable the use of USB and CD drives, but use cryptzone to ensure data is encrypted for privileged users who are allowed access.
  • ForeSee is working to implement a formal DLP solution in the future as part of the FedRamp compliance efforts.

Employee Computer Use

  • Using VPN to remotely access the company network should only be done from a company device. Home computers can be used if connecting through “View” (VMWare).
    • Employees must submit a ticket for initial access to the VPN, and provide the following information:
      • Personal cell number
      • Carrier
      • Personal email address
    • Once you have been given access to the VPN, Tech Ops can help you through the initial setup process.
    • To access the VPN, you must sign in through the browser (IE) at sslvpn.foreseeresults.com. You will be send a text message with a six digit string that must be input into the web page that displays after you login.
  • All PCs, laptops, and workstations should be secured with a password-protected screensaver with the automatic activation feature set at 15 minutes or less.
  • Employees should secure their workstations by logging off or locking when the workstation is attended.
    • Windows users can press Ctrl+Alt+Delete keys simultaneous and select Lock this computer, or press the Windows key together with L.
  • Employees must use extreme caution when opening email attachments received from unknown senders, which may contain viruses, email bombs, or trojans.
  • Access to the company’s email through a smartphone, using its native email client, is only allowed on a company issues phone. Users without a company issues phone may use their personal phone’s web browser to access company email if supported.
  • The employee must not install third-party file sharing tools, such as Dropbox, on a company device, without the consent of the Security Team. If permission is granted, the file share can only be used for non-proprietary data. No client data should be shared.
    • Contact the Security team for more information on the formal exception process.
  • Use encryption of information in compliance with Security Team standards:
    • Data classified as “public” does not need to be encrypted.
    • Data classified as “confidential” passed internally to an authorized user needs no encryption.
    • Data classified as “internal” does not need to be encrypted since they can only be sent internally.
    • Any other classification must have encryption.
    • WinZip is the standard internal encryption tool.

Personal Mobile Devices

Mobile devices may be used on the company public network only and should be done for valid business reasons only. The company supports Apple iOS and Android devices. An employee having a personal mobile Android or Apple device may connect to the company private network under the following conditions:

  • The device must be registered and staged by the IT staff.
  • The employee must consent to the device being cleaned and restored to the original factory settings upon the employee’s departure from the company and in the event the device is lost or stolen.
    • The employee must notify the company immediately if the device is lost or stolen.
  • The employee must consent to company-imposed password protection for the device, and the password must be kept confidential.
  • The company does not support any other type of mobile device, including but not limited to, the Windows or Blackberry operating systems.

User Security Groups

  • We will be categorizing clients as either HIPAA, FedRAMP, or neither. If you are an analyst supporting a client who is categorized as HIPAA, you must be in the HIPAA security group to be able to access that client’s data.
    • A user can have both HIPAA and FedRAMP clearance.
    • Users who have a specific designation are able to view data from all clients with that particular designation, not just data from your client(s) with that designation.
  • By default, ForeSee employees will be added to the ‘default’ security group, with no access to HIPAA or FedRAMP client data.
    • For employees of clients, there is no change to the Portal access process, since the client is designated at the company level as FedRAMP/HIPAA.
  • DOT is in charge of placing ForeSee employees within the correct security group.
  • Report subscriptions within the AAP will be affected by the addition of security groups.
    • If a subscription report is set up with an ‘owner’ that is not in the applicable security group the subscription will not process.

Data Classification Levels

ForeSee classifies data in five primary categories (Sensitive, Confidential, Restricted, Internal, and Public).

  • Sensitive Data: Any restricted data that is covered by law/regulation. The Company is required to self-report to the government and/or provide notice to the individual if information is inappropriately accessed. Regulations include data collected under FedRAMP, HIPAA, HITECH, FERPA, PCI and similar state and federal laws.
  • Restricted Data: Any user information which unauthorized disclosure could seriously and adversely impact the team member, company, business partners, and/or its customers. This is typically data not shared, such as passwords and encryption keys.
  • Confidential Data: any data that is contractually protected as confidential by law or by contract and any other information that is considered by the Company appropriate for confidential treatment. (Client-owned data, HR information, non-public personal and financial data).
  • Internal Data: Any information that is proprietary or business information which is intended strictly for use within the Company. (Internal policy, operating procedures, operational manuals, technical and security documents).
  • Public Data: Any information that may or must be made available to the general public, with no legal restrictions on its access or use. Corporate Communications approves any such designation.

The system/data owner, or stakeholder, is typically responsible for classifying the data. If you have any question surrounding which classification your data should be, contact the Security Team at security-team@answers.com.

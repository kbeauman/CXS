What Is It?

Respndent Level Integrate provides clients with the ability to either import and connect their own data with the ForeSee portal, or export and connect certain data from the ForeSee portal into their own reporting suite.

Why Use It?

Forward thinking organizations need to see how their customers’ experiences are impacting sales and ultimately shaping their business. ForeSee Integration - Respondent-Level can load information from various business systems into ForeSee adding context for decision making and providing a comparative format for detailed analysis. Deep, segmentation-level analysis can be performed to help guide strategies within specific customer segments, audience profiles, etc.

General Information

  • Typically, additional measures tied to the same integration type are free. Two or more separate data sources require multiple integrations, but ultimately the contract should always be checked to see how it was sold. In some cases, e.g., Adobe Omniture, we may have opted to charge per report suite.
  • Depending on the integration type, we can accommodate bi-directional passing of post-processed data. Please see the matrix under Important Links for product specific direction.
  • A custom API (provided by the clickstream provider) is only used in instances of Omniture or Google Analytics integration. For all other integrations, we must use a raw data file transfer via SFTP and EXDL process (the only exception is our unidirectional API option).
  • Cannot do clickstream integration on measurements focused on a non-Digital experience.
  • Mobile clickstream integration (i.e., Mobile Site and Mobile App and for Tablet and Smartphone) - We typically can not use the custom API solution for integrating. Instead, the client must be able to pass us the GA or Omniture ID as a CPP and we accommodate integration through an SFTP/EXDL process. CPPs are difficult to capture with an On-Exit Mobile survey solution. Consultation with IS is recommended for these scenarios.
  • QuestionID, answerID, and respondentID are all alphanumeric. This may cause some issues during the data transmission and append process for clients whose internal database is set up for numeric IDs only.

Requirements For Passing Data To ForeSee: Data File Requirements

  • The file must have column headers.
  • Delimited files (CSV, TSV, Pipe, or any other delimited format) with associated header record.
  • Any dates supplied in MM/DD/YYYY or M/D/YYYY H:MM format.
  • One row per respondent.
  • ForeSee respondent ID required as a data point in each row.
  • In most cases, Adobe (Omniture) or Google (UA) integrations are accommodated through an API those companies provide. When using this API, data file transfers are not necessary to deliver our data to Adobe or Google. However, if a client wants to pass Adobe or Google data back to us, a data file transfer is necessary.

Requirements For Passing Data To ForeSee: SFTP Information

  • ForeSee establishes an ID, Password, and directories to the SFTP host. A client-supplied SFTP can be used as well.
  • ForeSee also has the capability to provide Public Key Authentication.
  • Encryption with Open PGP and GnuPG (GPG) is supported provided the ForeSee encryption key is used.

Requirements For Passing Data To ForeSee: EXDL Information

External Data Load (EXDL) is a process used to load data into the ForeSee database:

  • A custom EXDL is generated for each data feed.
  • The process monitors the SFTP folder and automatically imports new files.
  • Imported data appears in the Portal the following day.

Requirements For Passing Data To ForeSee: Hidden Custom Questions Information

  • Hidden custom questions are simply the containers for the imported data. Respondents do not see these questions on their surveys.
  • Every incoming column of data is mapped to a corresponding hidden custom question.

Requirements For Passing Data From ForeSee

  • Data is in tab delimited files of Text (.txt), Comma Separated Values (.csv), or Excel (.xlsx) format.
  • One zip file per measure. The zip file contains three files:
    • Data file - Respondent data from the survey.
    • Question Mapping File - Question details including mapping.
    • Answer Mapping File - Answer details including mapping.
  • Delivery options include:
    • Email: Zip file attachment sent to email recipient.
    • Posted for Pickup: Posted to ForeSee Portal workspace area for client retrieval.
    • FTP (ForeSee): File available for retrieval on ForeSee SFTP site.
    • FTP (Client): File sent to client SFTP site.

Google Analytics Integration Information

No Cost Integration Option

In survey code versions prior to 18X (hosted code) we have the ability within our survey code to capture the Google Analytics cookies, parse the values and pass them along as CPPs. The captured values are:

  • source
  • campaign
  • medium
  • keyword

However, these cookies and values are ONLY available in the Google Analytics version PRIOR to Google Universal to which most of our clients have moved. If the client still has the Google Analytics version and our code version that accommodate this, we will not charge to collect these parameters. However, we should no longer “market” a free version.

Paid Integration Option

Google Analytics integration allows ForeSee to use an Application Programming Interface (API) to retrieve Google Analytics (GA) "dimension" information from our clients' GA accounts and append it to their existing respondents.

  • Data points needed to complete a successful integration:
    • Username
    • Password
    • ProfileID (UA number)
  • There is no limit to the type of dimensions that can be sent back to ForeSee, though Google’s API limits the number of dimensions sent over to seven.
  • Data transfer is uni-directional into ForeSee only.
  • Universal Analytics is API only. All legacy Google Analytics accounts will be auto migrated at some point to Universal Analytics.

Omniture/Adobe Integration Information

  • Minimum of 300 responses to show Sat scores in the portal. This applies to the clickstream program as well.
  • Omniture guarantees an 80% match of integration data.
  • Respondent ID n count may not match the Portal n count. This is because the respondent ID appears in Omniture before the survey has been processed on the ForeSee side.
  • The $10K charge applies to both the API and extract methods.
  • Prior to the start of ForeSee integration, a review of the Adobe Genesis ForeSee content must be performed in the Adobe Digital Marketing Suite:
    • Implementation guide
    • Videos
    • Wizard
    • ForeSee selections confirmation worksheet

      Click here for a video tutorial for Omniture Wizard setup.

  • Client needs to have the correct Adobe service level to accommodate integrations.
  • Omniture API for Enterprise Measures requires the same reporting suite endpoint.
  • When a client is hosting via Ensighten or DTM, they must whitelist our domain in order for the calls to go through without being blocked. They must also provide us with the code to call their Ensighten or DTM setup.
  • While it is technically possible to integrate on one eVar (respondentID), it is highly discouraged. Do not bring the matter up to a client unless they ask the question first.
  • Data transfer for Omniture API is bi-directional, Omniture non-API is uni-directional into Omniture only.
  • The following types of data may be passed to ForeSee:
    • Latent level Element scores and Satisfaction scores
    • CQ responses
    • CPPs
    • Up to 15 ForeSee values passed into SiteCatalyst
  • The following data may be passed to ForeSee (maximum values are per category):
    • Up to ~30 standard Adobe Omniture variables
    • Up to five traffic variables (sProps)
    • Up to five conversion events (eVars)
  • One ForeSee measure integrates into one SiteCatalyst report suite. A single ForeSee measure cannot integrate into multiple SiteCatalyst report suites.

During the integration process, the following activities must be completed:

  • By Client:
    • S-Code - Sent to ForeSee.
    • Report Suite - Sent to ForeSee.
    • Respondent ID/eVar - eVar must be created in SiteCatalyst and communicated to ForeSee.
    • Wizard completion - MID is required to start and complete the wizard.
  • By ForeSee:
    • MID - ForeSee must provide the client with the MID for the wizard setup.
  • By both the client and ForeSee:
    • Survey review - A review of your existing survey during which you identify which ForeSee variables are to be passed to Omniture.
    • Omniture Variables Review - The review of 30 standard variables, up to five sProps and five eVars.

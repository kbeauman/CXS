Advanced Analytics Portal Reporting Tab

The Reporting tab within the APP offers the following functionality:

  • Run Reports - Displays the Available Reports page where you can run On-Demand reports and select Subscription reporting.
  • View Filters - Displays the Search Filters page where you manage data filters.
  • View Jobs - Displays the Search Jobs page which lists all report requests within the selected time frame.
  • View Subscriptions - Displays the Search Subscription page where you manage Subscription reports.
  • View Distribution Lists - Displays the Distribution Lists page where you manage email addresses of report recipients.
  • Ad Hoc Reporting - Opens a JasperSoft session in a new browser window. See JasperSoft Help for information on using this feature.

How To Schedule Automated Reports in Advanced Analytics Portal

  1. Select the Subscription Reporting icon for the relevant report.
  2. Complete the parameters for this report, each report parameter page varies.
  3. Select Schedule.
  4. Complete the scheduling parameters for this report, entering the Recurrence Pattern, Range of Recurrence, Additional Options, and Email Options.
  5. Select Schedule.

The subscription report does not run until the next scheduled delivery date (as defined in the schedule parameters). When the report runs, it is delivered as defined in the schedule parameters and the subscriber receives an email notification.

Available Reports Within Advanced Analytics Portal

Comparison Reports

  • Benchmark Reports - A comparison of scores against peers, competitors, and best-in-class:
    • Cannot be run with a filter.
    • Prior 15 months of data can be accessed.
  • Segment Comparison - Compares Element, overall Satisfaction, and Future Behavior scores across a custom question:
    • Can be scheduled to run automatically for a previous day, week, or month.
    • Can be run for one question or all closed ended custom questions.
    • Can be run for a specific filter.
      • Available only for those with a service level that includes the filtering capability within the Portal.
    • Client to determine frequency with which they receive this report as well as which questions are included.
  • Cross Tab - A cross reference of selected questions which allows you to break down a single score.
    • Can be scheduled to run automatically for a previous day, week, or month.
    • Can be run for one closed-ended custom question crossed with another custom question.
    • Can be run for a specific filter.
      • Available only for those with a service level that includes the filtering capability within the Portal.
    • Client to determine frequency with which they receive this report as well as which questions are included.
  • Trend Line Report - Tracks Elements, Satisfaction, and Future Behaviors over time:
    • Can be scheduled to run automatically for a previous day, week, or month.
    • Can be run for one closed-ended custom question crossed with another custom question.
    • Can be run for a specific filter.
      • Available only for those with a service level that includes the filtering capability within the Portal.
    • Client to determine frequency with which they receive this report as well as which questions are included.
  • Benchmark True Conversion - A break down of participant intent for visiting the site. This report lists percentages of those who visited the site with the intent of purchasing, researching, or other, and of those, what percentage purchased.

Distribution Reports

    • Can be scheduled to run automatically for a previous day, week, or month.
    • Can be run for one question or all model questions.
    • Can be run for a specific filter.
      • Available only for those with a service level that includes the filtering capability within the Portal.
    • Client to determine frequency with which they receive this report as well as which questions are included.
  • Custom Question Distribution - The proportion of answer choices for custom questions:
    • Can be scheduled to run automatically for a previous day, week, or month.
    • Can be run for one question or all closed ended custom questions.
    • Can be run for a specific filter.
      • Available only for those with a service level that includes the filtering capability within the Portal.
    • Client to determine frequency with which they receive this report as well as which questions are included.

Reporting Within The Executive Portal

The Executive Portal does not have the same type of automatic reporting capabilities as AAP. Within the EP, dashboards function as reports, with the ability to create custom dashboards and then share them, either as a link that opens up a web page or a PDF image of the dashboard.

Shareable Link - The recipient can view the dashboard as a web page. However, the data does not change from the moment you click the Share icon.

PDF Download - The recipient can save the dashboard as a PDF document. Each dashboard tile is placed on a separate page with all the metadata displayed, i.e., the header is expanded. However, drill-down data is not available in a PDF.

Executive Portal Dashboards

Customization of the Dashboard for sharing purposes

  • A Dashboard is a collection of widgets called Tiles, which display a rating based on selected data types, such as demographics, environment, or customer satisfaction. There can be any number of tiles in a dashboard, and any number of dashboards.
  • Colors of widgets cannot be customized.
  • Tiles are comprised of customizable graphics, such as charts, grids, and Open Ends. The type of data displayed in a given tile originates from one of the following sources:
    • Scores - A Score is a numeric value, ranging from zero to one hundred, which represents a specific measure.
    • Benchmarks - Benchmarks are scores derived from other ForeSee clients for the purpose of comparison.
    • Open Ends - Open ends is a category of responses comprised of narratives from respondents.

Reporting Within Feedback

  • In-page reporting:
    • Access real-time collection results for specific site pages (and its child pages); also access site-wide metrics.
    • Access data via a URL extension – authentication required.
  • Reporting content:
    • Overview: Overall rating by filter criteria. Number of feedback surveys collected. Rated topics, Rating trends.
    • Insights: Ratings breakdown (counts and percentages) for codified answers.
    • Respondent Level: Respondent answers to all questions + open-ended comments + meta data.
    • Ability to define specific date/date range for reports.
  • Alerting:
    • Set-up email (no SMS) alerts based on criteria tied to a star rating, a topic or both.
      • Star Rating criteria can either be <, >, or = a specific star value.
      • Topic criteria can be = a specific topic.
      • Cannot have multiple conditions of the same criteria type (e.g., two different Star Rating criteria).
      • Can have conditions for two different criteria types, though the Boolean operator can only be “and”.
    • Email notifications sent to an individual, multiple individuals or a group alias.
  • Export and PDF versions of different reports.
  • Extract respondent data for selected periods into Comma Separated Values file.

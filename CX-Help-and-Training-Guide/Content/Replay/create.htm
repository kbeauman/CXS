Create a Replay Project

A single Replay project may be used in any number of Feedback Survey projects. The process of creating, editing, and managing Replay projects for use with Feedback is fundamentally the same:

  1. Click Create in the top navigation bar. The Create page displays with available project options based on your CX Suite subscription.
  2. Click Create Replay. The Create New Replay Project page displays.
  3. Type a descriptive identifier in the Title text box, a brief (80 characters maximum) narrative of the project in the Description text box and click Let's go. The Site Settings tab displays with the project title and the alert icon () to inform you the project must be published in order to activate the project, or apply any revisions to the project.
  4. By default, the project is turned off and must be turned on. Click the Tracking Is toggle switch to ON. The replay icon () and the Save button become enabled.
  5. (Optional) You may prevent the recording of select pages by clicking Do not monitor the following page URLs and entering the web page addresses of those pages.
    You may have no desire to record a user's interaction with a page of text, such as policies, case studies, or instructions. You can suspend the recording of these pages by adding the URLs to the Black List.
  6. Click Save. The Save button becomes disabled.
  7. Click Advanced Settings in the progress bar. The Custom PII Masking page displays.
  8. The Custom PII Masking page is intended for users with advanced programming knowledge to enable the masking of select user interface elements that may contain Personal Identity Information (PII). Type in the desired user interface elements under "pii": {. See ForeSee Replay Security for additional information on PII masking.
  9. Click Publish in the progress bar. The Publish Your Project page displays.
  10. Click the Publish button in Step 1. Even if there is no staging environment (server), this is a necessary first step. When the process completes, the Publish button in Step 2 is enabled.
  11. Select all syntax in the Staging Embed Snippet pane and paste the syntax into the <header> section of each web page in Staging to test the survey settings.
  12. Once testing is complete, click Publish to Production. When the process completes, a time stamp displays and Publish to Production becomes disabled.
  13. Select all syntax in the Production Embed Snippet pane and paste the syntax into the <header> section of each live web page. This is only necessary if a code snippet does not currently exist. For example, if you are adding Replay to an existing Feedback project, you can skip this step.
  14. Click either Projects or the ForeSee logo in the top navigation bar. The Projects page displays.
  15. Click the edit icon () of the Feedback project to have Replay. The Create/Edit Survey page displays.
  16. Click Configure Survey in the progress bar. The Configure Survey page displays.
  17. In the ForeSee Replay section, select the Yes radio button and click Save.
  18. Click Publish in the progress bar and publish the survey according to the Feedback Help instructions.
  19. Repeat the edit and publishing of each Feedback project to have Replay.

What Is It?

Created as a complement to ForeSee’s continuous customer experience measurement, ForeSee Feedback is a self-service solution allowing organizations to gather immediate feedback on a customer’s digital channel experience and quickly route it to the appropriate customer experience team.

Available in these channels:

  • Web
  • Mobile (site only)

Why Use It?

Companies that gather feedback gain valuable and actionable input that improves both their customers’ experiences and the financial health of the company by driving higher conversion rates and reducing costs.

ForeSee’s new solution revolutionizes how marketers gather and analyze feedback. Practitioners can build surveys and implement digitally without IT involvement. Marketers and analysts can use our in-page analytics to gain insights without having to leave their own website.

Survey Creation & Management

  • Build own survey - Edit on demand - Publish on demand - Cease on demand.
  • Allows existing surveys or survey questions to be copied and reused in other surveys - which also allows for the ability to revert to a previous version of the survey at any time.
  • Editing a survey that is already published generates a Work In Progress (WIP) version of the survey. That WIP version is stored in a draft state until those edits are ready to be published.
  • Create a branded identity for the organization seeking site visitor feedback.
  • Create a header; include a high-resolution banner.
  • Include a feedback badge/icon to draw attention and activate feedback participation.
  • Create a thank you message to appear after participants submit feedback.
  • Preview the survey inside the Survey Builder.
  • Feedback supports the client’s custom templates:
    • All templates must be approved and developed by Product and Technology.
    • Once a template is approved it is included in the drop down menu within the Survey Builder.

User Roles & Capabilities

  • Current roles are
    1. Create/Edit/Publish - Can build, revises, and publish surveys, as well as view all reporting.
    2. View Only - Can view surveys and see reporting.
  • A single user cannot have access under both roles.
  • No limit to the number of users who have access to a specific role.

Configurations

  • Question Types Available: 5-star ratings, radio buttons, multiple-choice checkboxes, drop-down lists and open-ended text boxes.
  • Define and present questions dynamically based on chosen topic.
  • Feedback badge can be oriented horizontally or vertically and placed at the top, middle, or bottom of the left or right side of a page to draw attention to solicit feedback participation.
  • Feedback badge can be customized by color, either by selecting from a suggested color palette or by typing in a custom hexadecimal code.
  • Badge can appear statically on a page or it can float with a scroll.
  • Badge appears in same location on the site/page regardless of device for responsive design sites.
  • Assign questions to be asked only on specific domain pages or entire domain, i.e., white listing.
  • Feedback survey can be presented in a pop-up window or a window overlay.
  • Clients have the ability to create custom privacy policy wording/name and link it to the URL of their privacy policy.

Deployment & Mondification

  • Publish feedback survey to test environment.
  • Edit and re-publish feedback survey as necessary.
  • Deploy a single survey or multiple feedback surveys within a single domain.
  • Feedback surveys only work on pages with hosted code.
  • Shut-off feedback survey with single switch.
  • Also offers a JavaScript Call Function to launch the survey manually. Allows clients flexibility such as adding the feedback survey to the page footer, create a custom badge and create event triggers. The client needs to understand how to work with JavaScript to utilize this feature - as we do not support this effort.

Reporting

  • In-page reporting:
    • Access real-time collection results for specific site pages (and child pages); also able to get site-wide metrics.
    • Access data via a URL extension – authentication still required.
  • Reporting content:
    • Overview: Overall rating by filter criteria, number of feedback surveys collected. Rated topics, rating trends.
    • Insights: Ratings breakdown (counts and percentages) for codified answers.
    • Respondent Level: Respondent answers to all questions + open-ended comments + meta data.
    • Ability to define specific date/date range for reports.
  • Export PDF versions of different reports.
  • Extract respondent data for selected periods into .CSV file.

Alerting

  • Set-up email (no SMS) alerts based on criteria tied to a star rating, a topic, keyword, or URL.
    • Star Rating criteria can either be <, >, or = a specific star value.
    • Topic criteria can be = a specific topic.
    • Cannot have multiple conditions of the same criteria type except for Keyword criteria. An alert can have more than one Keyword criteria (e.g., two different Star Rating criteria).
    • Can have conditions for more than one different criteria types, though the Boolean operator can only be “and”
  • Email notifications sent to an individual, multiple individuals or a group alias.

Metadata - Web

The following metadata is automatically captured for feedback responses on web devices (desktop/laptop):

  • Browser (e.g., Chrome 49)
  • Browser Version (e.g., 49)
  • Environment (e.g., staging or production)
  • Survey Presentation (e.g., desktop or mobile)
  • Browser name (e.g., Chrome)
  • Code Version (e.g., 1.0.11)
  • Respondent ID
  • WURFL ID

Metadata - Mobile

The following metadata is automatically captured for feedback responses on mobile devices (phones/tablets):

  • Brand Name (e.g., Google)
  • Browser Name (e.g., Chrome Mobile)
  • Browser (e.g., Chrome Mobile 30)
  • Browser Version (e.g., 30)
  • Code Version (e.g., 1.0.11)
  • Dual Orientation (e.g., True)
  • Is Tablet (true or false based on if the device is a tablet)
  • OS (e.g., Android 6.0)
  • OS Version (e.g., 6.0)
  • OS Name (e.g., Android)
  • Resolution
  • Resolution Width
  • Resolution Height
  • Screen
  • Screen Width
  • Screen Height
  • WURFL ID
  • Device (e.g., Android)
  • Environment (e.g., staging or production)
  • Model Name (e.g., Nexus 5)
  • Pointing Method (e.g., touchscreen)
  • Respondent ID
  • Survey Presentation (e.g., mobile)

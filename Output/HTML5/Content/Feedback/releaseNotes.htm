Release Notes

Spring 2016

Navigation changes to Insights

The following changes have been made in the top navigation bar and secondary bar:

  • Insights is now Analytics
  • Overview is now Summary
  • Questions is now The Survey

Survey Configure Flow Updates

We have made updates to the configure flow to make it easier for you to build, choose, configure, and publish your surveys.

You now have a new step to Choose Survey Type. This step really breaks out the survey badge and type customization from the configure flow. The Manual Launch Survey option (customer defined link) is now also a part of this page.

The Configure Survey step allows you to use the White List feature as you had in the past and take advantage of a new feature to update the Privacy Policy, which is described below.

Badge Customization

We are excited to let you know you can now update the color of the Feedback Badge from within the Choose Survey Type section of the Portal.

Updates:

  • You can now choose from a Suggested Colors swatch OR add you own hex value to match your brand.
  • Badge updates display in the right model as it did previously to reflect settings.
  • We have updated the badge on/off switch to now be Active or Inactive.

    Setting the badge to Inactive allows you to use the Manual Launch (customer defined link). This link had been previously found on the Publish page.

Privacy Policy Updates

This feature allows you to update the Privacy Policy link on your survey to link to your policies if you wish.

Updates:

  • From within the Configure Survey tab, you are now able to update the text and link to the privacy policy found within the survey modal.
  • You can examine these updates with the Preview feature of the Create/Edit Survey page.

 

Spring Follow-up Release

The Product and Engineering teams rolled-out the Spring Follow-up Release with the last few planned enhancements and a handful of issue fixes.

The Replay for Feedback capability is now available on the production servers. We plan to do a few rounds of internal testing with ForeSee.com, and will then make Replay available to a couple clients that have already purchased Replay, being careful to step them through the integration process.

In addition to adding Replay, this release of Feedback includes:

  • Surveys (within the Feedback product) may now contain hyperlinked text in the question. When editing a survey, you can highlight a character, word, or phrase, and a Link dialog displays to click and the link address to.
  • Users with administrator permissions may delete Feedback survey Replays, as well as the entire respondent record.
  • Custom work for clients was completed.
  • Other minor enhancements and bug fixes.

Like with all releases there are a few items that we carry over. Below are a list of known issues we are still working to address in additional:

  • Replay for Feedback support for Internet Explorer 9.
  • Respondent Meta Data includes the Trigger Code URL.

Winter 2015

Alert Updates

We have launched new alerting capabilities for our real time alerting system for Feedback in the VOC Portal. These are additional capabilities to the current system where you can set up alerts in the Portal based on the surveys you have created.

Keyword - We have added two ways to alert by Keyword:

  • Exact Match (=) is exact match (case/spelling everything exact)
    If your consumer has completed a feedback survey and ANYWHERE in an open-ended response this keyword direct, you receive an alert.
  • Contains is a partial (fuzzy match) keywords for English open-ends.
    If the keyword was "ForeSee" then "ForeSee" and "Foreseee" would both match. The search is also case insensitive. If your consumer has completed a ForeSee Feedback survey and ANYWHERE in an open-ended response this keyword, you receive an alert.
    • Phase 2 (to launch in Feb) of this will be to enhance language detection, remove common words, and expand fuzzy search.

URL Based - We have added two ways to alert by URL:

  • Exact Match (=)
  • Supports white-listing the same as before when you configure your survey.
    If consumer provides feedback on the exact URL or a white-listed URL designated, that triggers an alert.
Search Functionality for Respondents

This feature makes it easier for you to filter out certain information from your respondent data opening up the ability to drill down to any segment of feedback providers by any keyword in an open-ended comment, attribute data like browser or operating system, etc.

The Respondent page has been enhanced to include a search field so user can search on things like:

  • Respondent ID (you can grab this from an alert email or report to drill down to see the date in the portal)
  • Survey information (like keywords within comments)
  • Meta data (Browser, OS)

Fall 2015

New Features
  • Custom questions are now displayed on Insights pages instead of the default text.
  • The Project Description may now be edited after the project has been created.
  • Open End responses may now be expanded by clicking anywhere within the response.
  • Clicking on the Project Name now displays the Project Edit page.
  • When a Feedback survey displays as a pop-up, a respondent may only close the survey by clicking the traditional Windows Close icon. When a survey is closed, any entries made are saved to allow the respondent to resume the survey.
  • The words “Copy of” are now automatically added to the badge label when a project is duplicated.
  • The display for an empty state, i.e., No Records Found, has been updated.
  • The Projects menu is now dynamic to user permissions.
  • The default text in text fields is now hidden when the user clicks into the field.
  • The default text for Question Types and Topics has been updated.
  • The title of the Customize Survey tab has been changed to “Create/Edit Survey”.
  • Error messages now display near the Submit Feedback button to ensure respondents notice the message.
  • A Credit Card Masking Feature has been added to Feedback to obscure credit card numbers.

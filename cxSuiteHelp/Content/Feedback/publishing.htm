Publishing a Survey

Creating a Feedback Survey > Collection Methods > Publish a Survey

Going live with a survey is a two-step process where you first migrate the survey to a sub-environment, such as Staging or QA. This allows you the opportunity to review and test the survey before going live. Even if you do not have such an environment, you must perform this initial step. Then publish to Production to display the Feedback Badge on your web site. You may also set up Alerts from this page, which is optional.

The Publish tab offers an optional code snippet for each environment which must be embedded into each of your website templates after you publish. There is also a JavaScript Call Function in the Launch This Survey section to launch the survey from a menu, button, or other user interface element other than the Feedback Badge.

The Publish page can be accessed from the Projects page. Click the Edit icon () of the desired project and then click Publish.

When a project has revisions which have not been published, the Edit icon is replaced with an alert icon (), which functions the same way.

Publish Your Project

You can use the toggle switch This project is to turn the survey off or on. See Turn a Survey Off for additional information.

To publish a survey:

  1. If you are publishing a new survey, click Get the code. The Your Embed Code dialog displays.
    1. If you do not have a Staging, QA, or other preliminary environment (server), skip this step. In the Staging Code pane, click Copy to clipboard and paste the syntax into the <header> section of each web page in Staging from which to launch the survey in order to test the survey settings.
    2. In the Production Code pane, click Copy to clipboard and paste the syntax into the <header> section of each live web page from which to launch the survey.
    3. Click I'm done! The dialog closes.
  2. Click the This project is toggle to On. This toggle turns the Feedback Badge off or on and you must complete the publishing process before this setting takes effect.
  3. Click the Publish button in Step 1: Publish to Staging. Even if there is no preliminary environment (server), this is a necessary first step. When the process completes, the Publish button in Step 2: Publish to Production is enabled.
  4. Once testing is complete in Staging, click Publish to Production. When the process completes, a time stamp displays and Publish to Production becomes disabled.
  5. Go to the targeted web page(s) and verify the badge displays according to your settings. If the page was open before you published, you may need to refresh the page.
  6. Optionally, click Set up Alerts to set up notifications for the project. See Alerts for additional information.

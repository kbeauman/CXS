Collection Methods

Create A Project > Creating a Feedback Survey > Collection Methods > Publish a Survey

Use the Collection Methods page to manage how your survey is presented to the end-user. When editing or creating a survey, click the Collection Methods link near the top of the page to display these settings.

The Collection Methods page offers the following options:

  • Web Link - Use a hyperlink for email invitations, link on your social media page, etc.
  • Site Badge - Use a badge, button, or image to launch the survey.

Web Link

Using a web link is, perhaps, the quickest way to start collecting responses. This collection method requires you to create a Feedback survey, configure the URL settings (which is done on this page), and publish. The link is offered during this final step and can be copied and pasted as needed.

You can set the link to deactivate on a specified date, and determine the message that displays to visitors who click the link after the set date:

  1. Click the Edit icon () for Web Link. The Web Link page displays.
  2. If the Never option is selected for This Link Deactivates, click On. A date field and text box displays.
  3. Click in the date field to display a calendar to select the deactivation date.
  4. In the text box, type in the message you want to display to those who click the link after this date and click Save Settings. The Collection Methods page displays.

Site Badge

The Site Badge collection method allows you to set how and where respondents access the survey, as well as how the survey is presented. Click the Edit icon () for Site Badge to display the Badge Setup page. The following setting groups are available on the left rail:

  • Design Options - Set badge design and which device types to show the badge on.
  • URL Targeting - Set which web pages display the badge, as well as what topics.
  • Replay Options - Turn ForeSee® Replay off or on for the project. This only displays for clients with a Replay contract.

Design Options

Settings for each device type are the same. Select Hide to copy a code snippet for either a button or link deployment method.

The Click here to provide feedback buttons are for demonstration purposes only and non-fuctioning.

Select Show to set the following options:

Once a respondent completes a survey, the badge is hidden from your web page to prevent multiple submissions during a single visit.
  • Badge Icon - Select the icon to display with the Button Text.
  • Choose Text - Type in the badge label.
  • Icon Position - Set the location of the badge on the screen.
  • Survey Delivery - A menu of options for how the survey appears to the respondent.
  • Scroll Behavior - When selected, the badge maintains position in the browser window regardless of page scrolling.
  • Badge Direction - Displays the badge vertically or horizontally.
  • Badge Color - Click to display a dialog with suggested colors to select and a text box to type in a hexadecimal color code.
  • Badge Animator - When selected, the badge reacts when moused over.
  • A Badge On My Web Site - This is the default setting for new surveys and displays a highly customizable, capsule shaped graphic for respondents to invoke the survey.

    Custom settings for the badge are displayed in the preview pane and include:

  • Manually Launch Survey - Displays a JavaScript to launch the survey using another graphical interface item, such as a Feedback link in the footer of the page. See Manual Launch of Survey for instructions.

URL Targeting

Configure Survey

The Configure Survey tab allows you to set specific URLs for each topic (White List) and set the link to your own Privacy Policy URL.

White List

By default, each topic is set to display on every page of your web site. Clear the Show Site-wide check box of a topic to open a text box where you can type/paste in the URL of each page the topic is to show on.

Your online store has clothing lines for men, women, and children, and topics with questions specific to each clothing line. For the Children topic, you may want to enter all URLs of site pages containing your children's line. Upon Publishing, shoppers only see the questions of the Children topic when they click the FeedBack Badge while viewing the pages you white listed.
  1. If your site displays the URL with both http and https, create the white list entry with an asterisk (*) before the domain. Example: *ABCcompany.com
  2. If your staging environment has different URLs than production, be sure to include both staging URLs and production URLs in the white listing section.
  3. If the ForeSee code is deployed across multiple country pages, be sure to specify which domain you expect the survey to appear on. Example: *ABCcompany.com/cart* instead of *cart*.
  4. Remove all slashes (/) from the end of a white list URL.

Once you have completed your White List, click Save. Your revisions take effect when you publish.

Privacy Policy

If your organization has a standing web page or online document containing your Privacy Policy, you can manage the link that displays at the bottom left of the survey:

  • Choose Privacy Policy Text - This text box defaults to Privacy Policy, but you can revise it as desired, e.g., Privacy Pledge.
  • Privacy Policy URL - Type/paste the web address of the page containing the policy.

Click Save when you have finished. The link is changed when you publish.
